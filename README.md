# Пояснения build.yml:

on.push.branches – сборка будет запускаться при каждом пуше в ветку main.

on.push.tags – если вы создадите тег, начинающийся с v (например, v1.0.0), после сборки будет создан релиз с прикреплённым EXE‑файлом.

runs-on: windows-latest – используется виртуальная машина Windows (так как нам нужен .exe).

actions/checkout – клонирует ваш код.

actions/setup-python – устанавливает нужную версию Python.

pip install – ставит зависимости из requirements.txt.

pyinstaller – собирает один исполняемый файл с флагами --onefile и --windowed (без консоли). Имя выходного файла MyApp.exe.

actions/upload-artifact – загружает полученный EXE как артефакт сборки. Его можно скачать со страницы Actions.

release job – запускается только при наличии тега. Скачивает артефакт и создаёт релиз с этим файлом.

## Проверка работы CI/CD
Сделайте коммит и пуш в main. Перейдите на вкладку Actions вашего репозитория на GitHub – вы увидите запущенный workflow. После завершения вы сможете скачать артефакт MyApp-Windows с собранным EXE.

Чтобы создать релиз, создайте тег и пушните его:
```
git tag v1.0.0
```
```
git push origin v1.0.0
```

- При пуше тега v1.0.0-beta → сработает первый шаг (-beta есть), создастся pre‑release.

- При пуше тега v1.0.0 (без -beta) → сработает второй шаг, создастся обычный релиз.

После этого автоматически запустится сборка, и по окончании на странице Releases появится новый релиз с прикреплённым файлом.
